= Changelog

* 2026-04-28
** update xref:api_specifications.adoc#rule-sch-allof[] and xref:api_specifications.adoc#rule-sch-oneof[]: guidelines on use of `discriminator` and `mapping`
** new rule xref:resources-document.adoc#rule-child-ent[] on representation of child entities
** update rule xref:api_specifications.adoc#rule-oas-descr[] with exception for `title` within `allOf` and `not` inline schemas
** update xref:events.adoc#rule-async-svctype[]: `service` should be prefixed with a reverse-DNS name
** new rule xref:events.adoc#rule-async-order[] on out-of-order and duplicate messages
** update rule xref:events.adoc#rule-async-cespec[]:
*** clarify that an Asynchronous Message is immutable and all functional data should be part of it (i.e. not exclusively in exchange protocol)
*** guidelines on context attributes were split off to xref:events.adoc#rule-async-ctxt[]
** new rule xref:events.adoc#rule-async-generic[] with guidelines on generic endpoints for Asynchronous Messages
** add a list of organizations that endorse this guide (xref:introduction.adoc#Introduction[])

* 2026-02-16
** new rule xref:api_specifications.adoc#rule-oas-doc[] on providing documentation in OpenAPI. This replaces the previous `[ext-docs]` rule.
** update rule xref:api_specifications.adoc#rule-oas-descr[]:
*** extended with all OpenAPI free-text documentation properties
*** a `summary` should now be present for each operation
*** for inline schemas of type `object`, a `title` is now recommended. For schema components, it shouldn't override the component name if present.
** update rule xref:api_specifications.adoc#rule-sch-oneof[]: changed allowed usage to ensure tooling compatibility (`object` subschemas with `discriminator`, subschemas with only `required` and `enum` subschemas)
** update xref:tracing.adoc#rule-trc-header[]: make UUIDv4 optional for trace ids and recommend randomness

* 2025-11-25
** new guidelines on xref:events.adoc#events[ Asynchronous messaging and events]
** new rule xref:tracing.adoc#rule-trace-w3c[] on the use of W3C/OTel tracing only within an organization's trust boundary
* 2025-09-24
** add rules about composition, inheritance and polymorphism in schemas using `oneOf`, `allOf` and `discriminator` in xref:api_specifications.adoc#rule-sch-allof[] and xref:api_specifications.adoc#rule-sch-oneof[]
** add linked documentation pages for each issue type in xref:problems/badRequest.adoc#bad-request[]
** improve xref:internationalization.adoc#internationalization[] guidelines:
*** clarifications on the `Content-Language` header
*** language choice may also be based on a user profile
*** update xref:internationalization.adoc#rule-int-negot[]: language priority using `q` param, matching regional language variants and fallback behavior
** (minor): apply https://github.com/OAI/OpenAPI-Specification/releases/tag/3.1.1[OAS definitions] for an OpenAPI (entry) document and OpenAPI description throughout the guide
* 2025-06-16
** recommend OpenAPI 3.0 and remove OpenAPI 2.0 throughout the guide
** new in xref:problems/badRequest.adoc#bad-request[ Bad Request Problem]: standardized `invalidInput` issue type
** update section xref:resources-document.adoc#section-identifier[]: describe potential compatibility problems with escaped slash values in path params
** update rule xref:resources-document.adoc#rule-id-name[]: always prefix `id` and `code` path param names with their resource type
** update description xref:statuscodes.adoc#http-409[`409 Conflict`] status: conflict with "data on the server" instead of "current state of the target resource". Remove optimistic locking issue (usually `412`)

* 2025-01-28
** new in xref:problems/badRequest.adoc#bad-request[ Bad Request Problem]: standardized `referencedResourceNotFound` issue type
** update of xref:errorhandling.adoc#standardized-problem-types[]: add `WWW-Authenticate` header for access token or scope problems
** update rule xref:internationalization.adoc#rule-int-negot[]: a default fallback language should be used when `Accept-Language` header can't be respected
